# Flappy Bird Clone — Design Spec

## Overview

A faithful Flappy Bird clone built with Phaser 3, using code-generated pixel art. Designed for publishing online, targeting both mobile (tap) and desktop (spacebar/click) equally.

## Game Canvas

- **Resolution:** 360 x 640 (portrait, 9:16)
- **Scaling:** `Phaser.Scale.FIT` with `CENTER_BOTH`
- **Renderer:** `Phaser.AUTO` (WebGL with canvas fallback)

## Scene Structure

Two scenes:

1. **BootScene** — preloads all assets (generates pixel art textures from code). No visible UI needed since assets are tiny and load instantly.
2. **PlayScene** — manages all game states internally via a state machine:
   - `READY` — bird hovers gently, "Get Ready" message displayed, waiting for first input
   - `PLAYING` — gravity active, pipes spawning, score counting
   - `GAME_OVER` — bird has collided, "Game Over" displayed with final score, waiting for input to restart

## Visual Style

**Combined style:** soft pixel art foreground with atmospheric background.

### Background (static, non-scrolling)
- Solid light blue sky (`#87CEEB`)
- Soft blurred cloud wisps that fade gradually from sky into cityscape — no hard horizontal cutoff
- City skyline with buildings at very low opacity (~40%), faint windows (~50% opacity) — reads as distant atmosphere, not solid objects
- Background buildings (behind foreground ones) even more faint (~25% opacity)
- Overall haze/atmosphere gradient layer tying sky into city

### Foreground (scrolling)
- **Bird:** Yellow body (`#f7dc6f`), white eye with dark pupil, red-orange beak, darker yellow wing. Softer dark outlines (`#2c3e50`), slightly rounded corners. ~34x24px sprite, 3-frame wing animation (up/mid/down).
- **Pipes:** Green gradient (dark edges `#73bf2e`, lighter center `#82d636`), dark green outlines (`#2d5a1b`). Cap/lip slightly wider than body. ~40px wide body, ~48px wide cap.
- **Ground:** Green top edge strip, then repeating tan/sandy stripe pattern (`#ded895` / `#d4c87a`), brown border top (`#8B7355`). ~78px tall. Scrolls continuously.

### Asset Architecture
Each visual element is generated by an independent function that produces a Phaser texture by key name:
- `bird` — 3-frame spritesheet texture
- `pipe-body` — rectangular pipe body texture (used for both top and bottom, top pipe's body is flipped vertically)
- `pipe-cap` — wider cap/lip texture (placed at the opening end of each pipe)
- `ground` — tileable ground strip
- `background` — full background with sky, clouds, city

Assets can be replaced with PNG files by loading them under the same texture key in BootScene, without changing PlayScene code.

```
src/
  assets/
    bird.js        → generates 'bird' texture
    pipe.js        → generates 'pipe' texture
    background.js  → generates 'background' texture
    ground.js      → generates 'ground' texture
  scenes/
    BootScene.js   → calls asset generators
    PlayScene.js   → references texture keys only
  config.js        → game constants
  main.js          → Phaser game config + entry point
index.html         → host page
```

## Game Mechanics

### Bird Physics
- **Gravity:** ~1200 px/s²
- **Flap:** sets velocity Y to ~-400 (instant upward impulse, not additive)
- **Rotation:** bird angle follows velocity — nose up when rising, nose down when falling
- **X position:** fixed at ~25% from left edge (90px)
- **Hitbox:** slightly smaller than visual sprite (forgiving)
- **Top boundary:** bird is clamped and cannot fly above the top of the screen

### Pipes
- Spawn from right edge, move left at constant speed (~200 px/s)
- Each spawn creates a pair: top pipe (flipped) + bottom pipe
- **Gap between pipes:** ~130px
- **Gap Y position:** randomized within a safe range (not too close to top/bottom)
- **Spawn interval:** one new pair every ~1.5 seconds
- Destroyed when fully off-screen left
- **Object pooling** via Phaser groups to avoid garbage collection stutters

### Scoring
- +1 point when bird passes a pipe pair's center X
- Tracked via invisible trigger zone on each pipe pair
- Displayed as large white pixel-style text with dark outline shadow, top-center of screen

### Collisions & Death
- Bird hits any pipe → game over
- Bird hits the ground → game over
- On game over: bird stops, pipes stop scrolling, ground stops, "Game Over" text appears with final score, input restarts

### Ground Scrolling
- Ground texture tile-scrolls left at same speed as pipes
- Acts as physics body for collision detection

## Input

| Platform | Action |
|----------|--------|
| Mobile   | Tap anywhere on screen |
| Desktop  | Spacebar or left-click |

Same input used for flap (during PLAYING), start (during READY), and restart (during GAME_OVER).

## Game Constants

All tuning values in a single config object for easy adjustment:

```js
const CONFIG = {
  gravity: 1200,
  flapVelocity: -400,
  pipeSpeed: 200,
  pipeGap: 130,
  pipeInterval: 1500,    // ms
  birdX: 90,
  groundHeight: 78,
  gameWidth: 360,
  gameHeight: 640,
};
```

## Technical Stack

- **Framework:** Phaser 3 (loaded via CDN or npm)
- **Physics:** Arcade
- **Language:** JavaScript (ES modules)
- **Build:** None required — vanilla JS served directly. Single `index.html` entry point.
- **Deployment:** Static files, deployable to any hosting (Netlify, Vercel, GitHub Pages, etc.)

## Out of Scope

- High score persistence / leaderboards
- Sound effects / music
- Difficulty modes / progression
- Power-ups
- Medal awards
- Pause functionality
- Settings / options menu
